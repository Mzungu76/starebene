MILESTONE 1 — Bootstrap repository, UI base, autenticazione (email+password), protezione route

Obiettivo
Creare un’app Next.js completa di autenticazione con credenziali (email+password), sessioni, layout responsive desktop/mobile e route protette. Nessun invio email, nessun provider esterno.

Task tecnici

Setup progetto

Next.js (App Router) + TypeScript

Tailwind CSS

shadcn/ui installato e configurato

ESLint + TypeScript strict

Struttura cartelle:

app/

components/

lib/

prisma/

scripts/

docs/ (solo file statici)

README.md

.env.example

Layout e navigazione responsive

Desktop: sidebar sinistra + topbar

Mobile: bottom navigation (Dashboard, Oggi, Piani, Check-in, Impostazioni)

Componenti riusabili:

AppShell

Sidebar

BottomNav

PageHeader

Card/StatsCard

Tema pulito, tecnico, coerente, leggibile su mobile.

Autenticazione Credentials

Implementare Auth.js/NextAuth con Credentials Provider:

Registrazione: email + password + conferma password

Login: email + password

Logout

Password hashing:

argon2 (preferibile) o bcrypt (accettabile)

Validazioni:

email valida

password min 10 caratteri

rate limiting base su login (anche semplice, per IP o per email, con cooldown in DB)

Sessione:

cookie httpOnly

route /app/* accessibile solo con sessione valida

UI:

/login

/register

redirect a /app dopo login

redirect a /login se non autenticato

Recovery codes (reset senza email)

Alla prima registrazione, generare 10 recovery codes monouso:

mostrarli una sola volta in una schermata dedicata “Salva questi codici”

salvarli in DB come hash (mai in chiaro)

In /app/settings/security:

rigenerazione recovery codes (richiede password)

Flusso “recupero”:

pagina /recover: email + recovery code + nuova password

verifica hash, marca code come usato, aggiorna password hash

Criteri di accettazione

Registrazione/login/logout funzionanti in locale e su Vercel

/app bloccato senza login

Recovery codes generati, visualizzati una sola volta e funzionanti per recupero password

UI corretta su mobile e desktop

Build e typecheck passano
